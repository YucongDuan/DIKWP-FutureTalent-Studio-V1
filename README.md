# DIKWP FutureTalent Studio V1

Standalone, offline-first prototype for evidence-based recruitment, internal mobility, workforce transition, and development assessment in the AI transformation era.

## Core principle

The system does **not** make automated hiring, rejection, promotion, salary, discipline, or ranking decisions. It organizes role tasks, candidate evidence, structured interviews, development pathways, fairness controls, and human review.

## Main modules

1. Future Role Canvas
2. Candidate Consent & Rights
3. Evidence Ledger
4. DIKWP Talent Closure
5. Task-Evidence & Transition Map
6. Structured Interview Workbench
7. Human Review Committee
8. Development & Reskilling Plan
9. Fairness, Privacy & Governance Audit
10. Transformation Simulation Lab
11. Audit & Export

## Quick start

Open `index.html` in a modern browser. No server, database, API key, or internet connection is required.

Optional CLI:

```bash
python run_demo.py
```

## Prohibited uses

- automated hiring or rejection
- automated employee ranking
- firing, promotion, salary, discipline, or benefit eligibility decisions
- emotion recognition, facial analysis, voice-based personality scoring
- political, religious, health, union, family, or other sensitive profiling
- covert surveillance or social-media scraping
- using development assessments as blacklists
- removing human review from high-impact employment decisions

## Design lineage

This prototype extends the public DIKWP WorkBridge OS concept into a visual, standalone recruitment and development studio while preserving its human-review-first and development-oriented boundaries.
